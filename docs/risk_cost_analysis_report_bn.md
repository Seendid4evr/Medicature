# মেডিকেয়ার (Medicature): বিস্তারিত রিস্ক এবং কস্ট অ্যানালাইসিস রিপোর্ট

**কোর্সের নাম:** CSE495 IT Project Management and Entrepreneurship
**প্রজেক্ট:** Medicature (ডিজিটাল হেলথ প্ল্যাটফর্ম)
**সিস্টেম সাইজ:** ~৬,০০০ লাইন কোড (6 KLOC)

## ১. প্রজেক্টের সারসংক্ষেপ (Executive Summary)
এই রিপোর্টটি মূলত **Medicature** প্রজেক্টের একটি অত্যন্ত বিস্তারিত ব্যয় (Cost) এবং ঝুঁকি (Risk) বিশ্লেষণ। এটি বাংলাদেশের অভ্যন্তরীণ বাজারের জন্য তৈরি একটি ডিজিটাল হেলথ প্ল্যাটফর্ম যেখানে ২১,০০০+ ওষুধের অফলাইন ডেটাবেস, স্মার্ট পিল অ্যালার্ম এবং PWA (Progressive Web App) সুবিধা রয়েছে। সম্পূর্ণ রিপোর্টটি বাংলাদেশের বর্তমান সফটওয়্যার ডেভেলপমেন্ট বাজারের বাস্তবসম্মত খরচ এবং কোর্স ম্যাটেরিয়ালের (COST ANALYSIS & Risk) উপর ভিত্তি করে তৈরি করা হয়েছে।

## ২. কস্ট অ্যানালাইসিস (Cost Analysis)

### ২.১ খরচের শ্রেণীবিভাগ (Cost Categorization)
যেকোনো প্রজেক্টের বাজেট সঠিকভাবে নির্ধারণ করার জন্য খরচগুলোকে নিচের ভাগে বিভক্ত করা হয়েছে:
- **Direct Costs (প্রত্যক্ষ খরচ):** ডেভেলপারদের পারিশ্রমিক (আনুমানিক ৩ Person-Months), ক্লাউড VPS সার্ভার সেটাপ এবং ২১ হাজার ওষুধের বিশাল ডেটা প্রসেসিংয়ের খরচ।
- **Indirect Costs (পরোক্ষ খরচ):** প্রজেক্ট পরিচালনার জন্য ব্যবহৃত বিভিন্ন অ্যাডমিন টুলস, ইউটিলিটি বিল এবং টিমের মিটিং খরচ।
- **Opportunity Costs (সুযোগ ব্যয়):** তৈরি করা API সাবস্ক্রিপশন না কিনে, নিজেদের অফলাইন ডেটাবেস ডেভেলপ করার জন্য যে সময় এবং শ্রম ব্যয় হয়েছে।
- **Risk-Related Costs (ঝুঁকি-সংশ্লিষ্ট খরচ):** হঠাৎ করে সিস্টেমে আসা বাগ (যেমন Character Encoding / Mojibake সমস্যা) এবং ডেটাবেস আপডেট করার জন্য যে এক্সট্রা ইমার্জেন্সি ফান্ড (Contingency Reserves) ব্যবহার করা হয়েছে।

### ২.২ খরচ অনুমান (Analogous Estimating / Ratio Model)
আমরা বাংলাদেশের লোকাল আইটি মার্কেট ট্রেন্ডের ওপর ভিত্তি করে **Analogous Estimating** পদ্ধতি ব্যবহার করে প্রজেক্টের মূল খরচ বের করেছি:
- **বেঞ্চমার্ক নির্ধারণ (Bottom-Up Estimation Proxy):** বাংলাদেশের আইটি সেক্টরে সাধারণত ডেভেলপারদের "ম্যান-মান্থ" (Person-Months) হিসেবে খরচ হিসাব করা হয়। একজন মিড-লেভেল ডেভেলপার সাধারণত প্রতিদিন ৫০-১০০ লাইন প্রোডাকশন কোড লেখেন। সেই হিসাবে ১০ হাজার লাইন (১০ KLOC) কোডের একটি স্ট্যান্ডার্ড পোর্টাল বানাতে ২ জন ডেভেলপারের প্রায় ২.৫ মাস সময় লাগে (মোট ৫ Person-Months)। প্রতি মাসে একজন ডেভেলপারের গড় এজেন্সির খরচ ৯০,০০০ টাকা ধরলে, বেজলাইন কস্ট দাঁড়ায় `৫ Person-Months × ৯০,০০০ টাকা = ৪,৫০,০০০ টাকা` (অর্থাৎ প্রতি হাজার লাইন কোডে বা KLOC তে খরচ প্রায় ৪৫,০০০ টাকা)।
- **Medicature প্রজেক্টের সাইজ:** আমাদের কোডবেসে ঠিক **৫,৬০৫ লাইন কোড** (~৬ KLOC) রয়েছে (PHP, MySQL এবং Javascript ব্যবহৃত)।
- **৬ KLOC এর বেসিক খরচ:** `৬ × ৪৫,০০০ টাকা = ২,৭০,০০০ টাকা`। 
- **জটিলতা (Complexity Factor):** যেহেতু এই অ্যাপে স্পর্শকতর হেলথকেয়ার ডেটা এবং PWA অফলাইন সাপোর্ট রয়েছে, তাই এর উপর **১.২৫ (High)** Complexity মাল্টিপ্লায়ার ধরা হয়েছে।
- **সর্বমোট বেজলাইন বাজেট (BAC - Budget at Completion):** `২,৭০,০০০ টাকা × ১.২৫ = ৩,৩৭,৫০০ টাকা`। 

### ২.৩ অর্জিত মূল্য ব্যবস্থাপনা (Earned Value Management - EVM)
"আমরা কি বাজেটের চেয়ে বেশি টাকা খরচ করছি? আমরা কি রুটিনের চেয়ে পিছিয়ে আছি?" — এই বিষয়গুলো পরিষ্কারভাবে পরিমাপ করার জন্য প্রজেক্টের ঠিক অর্ধেক সময় (৫০% টাইমলাইন) পার হওয়ার পর আমরা EVM বিশ্লেষণ করেছি।

**প্রজেক্টের বর্তমান ডেটা:**
- **সর্বমোট বাজেট (BAC):** ৩,৩৭,৫০০ টাকা (অ্যাপ্রুভ করা বেজলাইন)
- **পরিকল্পিত কাজ (Planned Work):** ৫০%
- **অর্জিত বা কমপ্লিট কাজ (Actual Work):** ৫৫% (ডেটাবেস বাগ ফিক্স এবং ফ্যামিলি মডিউল ফুল কমপ্লিট)
- **প্রকৃত খরচ (AC - Actual Cost):** ২,১০,০০০ টাকা (বাগ ফিক্স এবং ডেটা রিকভারিতে কিছুটা অতিরিক্ত ব্যয় হয়েছে)

**EVM ক্যালকুলেশন:**
1. **Planned Value (PV) =** ৩,৩৭,৫০০ টাকার ৫০% = **১,৬৮,৭৫০ টাকা**
2. **Earned Value (EV) =** ৩,৩৭,৫০০ টাকার ৫৫% = **১,৮৫,৬২৫ টাকা**

**পারফরম্যান্স মূল্যায়ন:**
- **Schedule Variance (SV):** `EV - PV = ১,৮৫,৬২৫ - ১,৬৮,৭৫০ = +১৬,৮৭৫ টাকা` *(আমরা রুটিনের চেয়ে এগিয়ে আছি)*
- **Schedule Performance Index (SPI):** `EV / PV = ১,৮৫,৬২৫ / ১,৬৮,৭৫০ = ১.১০` *(আমরা পরিকল্পিত কাজের চেয়ে ১১০% দ্রুত কাজ সম্পন্ন করছি)*
- **Cost Variance (CV):** `EV - AC = ১,৮৫,৬২৫ - ২,১০,০০০ = -২৪,৩৭৫ টাকা` *(বাগ ফিক্স ও টেকনিক্যাল কাজের কারণে বাজেট কিছুটা বেশি খরচ হয়েছে)*
- **Cost Performance Index (CPI):** `EV / AC = ১,৮৫,৬২৫ / ২,১০,০০০ = ০.৮৮` *(আমরা প্রতি ১ টাকা খরচের বিপরীতে ০.৮৮ টাকার ভ্যালু জেনারেট করছি)*

**চূড়ান্ত খরচের পূর্বাভাস (Forecasting):**
- **Estimate at Completion (EAC):** `BAC / CPI = ৩,৩৭,৫০০ / ০.৮৮ = ৩,৮৩,৫২২ টাকা`
- **Variance at Completion (VAC):** `BAC - EAC = ৩,৩৭,৫০০ - ৩,৮৩,৫২২ = -৪৬,০২২ টাকা`
- *উপসংহার:* প্রজেক্টটি খুব দুর্দান্ত গতিতে এগিয়ে যাচ্ছে (SPI = ১.১)। যদিও প্রাথমিক বাজেটের চেয়ে প্রজেক্টটি শেষ হতে প্রায় ৪৬ হাজার টাকা বাড়তি খরচ হবে (EAC ৩.৮ লাখ টাকা), তবুও এর পারফরম্যান্স চমৎকার।

## ৩. ঝুঁকি বিশ্লেষণ (Risk Analysis)

### ৩.১ বিস্তারিত SWOT বিশ্লেষণ (SWOT Analysis)
বাংলাদেশের বর্তমান হেলথটেক মার্কেটের বাস্তব অবস্থার ওপর ভিত্তি করে।

**Strengths (শক্তি / অভ্যন্তরীণ সুবিধা):**
- **স্বকীয় ডেটাবেস অ্যাসেট:** ২১ হাজার ওষুধের শক্তিশালী অফলাইন ডেটাবেস যুক্ত করা হয়েছে। অন্যান্য অ্যাপের মতো আমাদের প্রতিটা ক্লিকে API কল করতে হয় না।
- **দারুণ আর্কিটেকচার:** পুরো সিস্টেম মাত্র ৬,০০০ লাইনের খুব গোছানো কোডের উপর দাঁড়িয়ে আছে। অ্যাপটি PWA হওয়ায় এটি ইউজারদের ফোনে লোকাল ক্যাশ করে এবং খুব ফাস্ট লোড হয়।
- **ফ্যামিলি হেলথ ওয়ালেট:** ஒரே জায়গা থেকে নিজের এবং পরিবারের অন্য সদস্যদের ওষুধের অ্যালার্ম ট্র্যাক করার দারুণ সুবিধা।

**Weaknesses (দুর্বলতা / অভ্যন্তরীণ চ্যালেঞ্জ):**
- **টেকনিক্যাল সমস্যা:** আমাদের লিগ্যাসি PHP ব্যাকএন্ডে ইতিপূর্বে Character Encoding (Mojibake) বাগ দেখা দিয়েছিল, যা সিস্টেমকে ক্র্যাশ করিয়েছিল। 
- **ডেটাবেস মেইনটেন্যান্স:** এখনও নতুন ওষুধের ডেটা যোগ করতে ম্যানুয়ালি CSV ইমপোর্ট করতে হয়। DGDA (বাংলাদেশের ঔষধ প্রশাসন অধিদফতর) এর সাথে কোনো অটোমেটিক সিঙ্ক সিস্টেম নেই। 

**Opportunities (সুযোগ / বাহ্যিক সম্ভাবনা):**
- **স্মার্ট অ্যালার্মের মার্কেটে গ্যাপ:** বাংলাদেশের বর্তমান স্টার্টআপগুলো (যেমন আরোগ্য বা ডায়বেটিক স্টোর) শুধুমাত্র ওষুধ বিক্রি আর ডেলিভারিতে আগ্রহী। "স্মার্ট পিল অ্যালার্ম" দিয়ে ইউজারের প্রতিদিনের স্বাস্থ্য অভ্যাস ট্র্যাক করার মার্কেটটি এখনো পুরোপুরি ফাঁকা। 
- **AI Triage এর সম্ভাবনা:** প্রচুর ইউজারবেস তৈরি হলে সহজেই এআই (AI) সিম্পটম ট্রায়াজ চালু করে টেলিকনসাল্টেশন ব্যবসায় নামা যাবে। 

**Threats (হুমকি / বাহ্যিক ঝুঁকি):**
- **ডেটা প্রাইভেসি আইন এবং কমপ্লায়েন্স:** রোগীদের ব্যক্তিগত স্বাস্থ্য সম্পর্কিত অত্যন্ত সেনসিটিভ তথ্য স্টোর করার কারণে **বাংলাদেশ সাইবার নিরাপত্তা আইন (CSA)** এবং ডেটা সুরক্ষা আইনের কড়া নজরদারিতে পড়তে হতে পারে।
- **টাকায় ভরপুর প্রতিযোগী:** ডকটাইম (DocTime) বা প্রভা হেলথের মতো লোকাল জায়ান্টদের বিশাল ইনভেস্টমেন্ট আছে। তারা যদি হঠাৎ করে তাদের অ্যাপে আমাদের মতো স্মার্ট অ্যালার্ম ফিচার নিয়ে আসে, তবে আমরা বড় চ্যালেঞ্জের মুখে পড়ব।

### ৩.২ প্রত্যাশিত আর্থিক মূল্য (Expected Monetary Value - EMV) 
প্রজেক্টের লঞ্চের পর সবচেয়ে সম্ভাব্য ঝুঁকিগুলোর সাথে আর্থিক অংক যোগ করে আমরা EMV ক্যালকুলেট করেছি: 

| ঝুঁকির দৃশ্যপট (Risk Scenario) | সম্ভাবনা (Probability - P) | ইমপ্যাক্ট বা ক্ষতি (Impact - I) [টাকায়] | রিস্ক স্কোর (EMV) [টাকায়] |
| :--- | :--- | :--- | :--- |
| **Risk A: ডেটা লিক হওয়া / Cyber Security Act মামলা (হুমকি)** | 0.30 | -30,000 | -9,000 |
| **Risk B: বড় প্রতিযোগীদের নতুন আপডেট ও প্রসার (হুমকি)** | 0.50 | -75,000 | -37,500 |
| **Risk C: লঞ্চের পর অ্যাপে সিস্টেম বাগ দেখা দেওয়া (হুমকি)** | 0.40 | -20,000 | -8,000 |
| **Risk D: লোকাল মার্কেটে ইউজারের বিশাল জনপ্রিয়তা (সুযোগ)** | 0.60 | +150,000 | +90,000 |
| **সর্বমোট প্রত্যাশিত আর্থিক মূল্য (Total EMV)** | | | **+35,500 BDT** |

*ব্যাখ্যা (Interpretation):* আমাদের সর্বমোট EMV পজিটিভ (৩৫,৫০০ টাকা)। ৬০% সম্ভাবনা আছে যে অ্যাপটি দারুণ মার্কেট ফিট পাবে, আর এই একটি মাত্র বড় সুযোগ আমাদের সকল টেকনিক্যাল এবং প্রতিযোগী সংক্রান্ত ঝুঁকিকে আর্থিক দিক থেকে কভার করে লাভজনক পর্যায়ে নিয়ে যাবে।

## ৪. উপসংহার (Conclusion)
খুবই সুন্দর এবং অপটিমাইজড কোডবেস হওয়ার কারণে (~৬,০০০ লাইন) Medicature প্রজেক্টটি খুব কম বাজেটের ভেতরেই (EAC ৩.৮ লাখ টাকা) এন্টারপ্রাইজ লেভেলের সুবিধা (PWA, 21k Database) দিতে সক্ষম। যদিও বাগ ফিক্সিংয়ের কারণে আমাদের কস্ট-এফিশিয়েন্সি (CPI) কিছুটা কমেছে, কিন্তু কাজের স্পিড (SPI = ১.১) এবং অত্যন্ত লাভজনক পজিটিভ রিস্ক প্রোফাইলের (EMV +৩৫ হাজার টাকা) কারণে এটি একটি দারুণ এবং সফল ভেঞ্চার হিসেবে দাঁড়াতে যাচ্ছে।
